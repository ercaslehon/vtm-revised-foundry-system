# Vampire: The Masquerade Revised - Foundry VTT System

Неофициальная система для Foundry VTT под Vampire: The Masquerade Revised / VtM v3 Revised.

Система ориентирована на домашние хроники, удобный чарлист вампира, встроенные справочники и быстрый доступ к основным игровым сущностям: кланам, дисциплинам, магии крови, ритуалам, достоинствам, недостаткам, дополнениям, Путям и Дорогам.

## Статус проекта

- Package ID: `vtm-revised`
- Текущая версия: `10.0.11`
- Foundry VTT: minimum `13.347`, verified `14`
- Автор: `Ercaslehon`
- Тип проекта: fan-made / unofficial system
- Основной язык интерфейса: русский
- Дополнительная локализация: английский

## Установка через Foundry

В Foundry VTT открой:

```text
Setup -> Game Systems -> Install System -> Manifest URL
```

Вставь ссылку:

```text
https://raw.githubusercontent.com/ercaslehon/vtm-revised-foundry-system/main/system.json
```

Foundry прочитает `system.json` и скачает zip из GitHub Release.

## Основные возможности

### Лист персонажа

- Чарлист вампира.
- Чарлист NPC.
- Вампирский интерфейс с вкладками `Основное`, `Силы`, `Профиль`.
- Портрет персонажа и просмотр большого портрета по клику.
- Основные данные персонажа: имя, клан, поколение, сир, концепт, Натура, Маска, Путь или Дорога.
- Поддержка истории, целей, заметок, внешности, имущества и связей.

### Характеристики и способности

- Атрибуты: физические, социальные, ментальные.
- Способности по актуальному листу проекта: Таланты, Навыки, Знания.
- Специализации.
- Светящиеся точки характеристик.
- Лимиты точек по поколению.
- Корректное отображение расширенных значений для старших поколений.

### Ресурсы и трекеры

- Кровь.
- Сила Воли.
- Человечность / Путь / Дорога.
- Здоровье.
- Ограничения поколения: максимум черт, максимум крови, трата крови за ход.
- Опыт: редактируемое значение `Доступно` и журнал опыта во вкладке `Профиль`.

### Журнал опыта

Во вкладке `Профиль` есть журнал опыта.

Поддерживается начисление опыта, трата опыта, ручная корректировка, причина операции, покупка или изменение, пользователь, дата операции, удаление записи и пересчёт итогов по журналу.

### Броски

- Броски d10.
- Одиночные и комбинированные броски.
- Ручная сложность.
- Учет штрафов от здоровья.
- Поддержка логики `10 = 2 успеха` и `1 = -1 успех`.
- Чат-карточки бросков в читаемом стиле.

### Дисциплины и силы

- Старшие уровни дисциплин.

- Поддержка редких дисциплин и линий крови, включая Демонизм, Шаманство, Полёт, Сангвинус, Бардо, Мельпомению, Танатозис, Мистерию, Обеах и Валерен.

### Магия крови и ритуалы

- Пути магии крови.
- Ритуалы.
- Карточки путей.
- Карточки ритуалов.
- Светящиеся точки путей и ритуалов.
- Поддержка Тауматургии, Некромантии, Темной Тауматургии, Чародейства Ассамитов, Чародейства Сеттитов и других справочных путей из каталогов.

### Мастер создания персонажа

Пошаговый мастер создания персонажа помогает собрать базовую структуру персонажа и выбрать ключевые элементы: клан, Путь или Дорогу, справочные карточки и стартовую настройку персонажа.

### Автоматический импорт справочников

При первом запуске мира GM-ом система автоматически импортирует встроенные справочники:

- кланы и линии крови;
- дисциплины;
- магия крови;
- ритуалы;
- оружие;
- достоинства и недостатки;
- дополнения;
- Пути и Дороги.

Повторный запуск не должен плодить дубли. Для этого используется служебная настройка мира `catalogSeedVersion`.

Ручной запуск автоимпорта:

```js
game.vtmRevised.seedBuiltInCatalogs()
```

Ручной импорт всех встроенных каталогов:

```js
game.vtmRevised.importBuiltInAllCatalogs()
```

Отдельные команды импорта:

```js
game.vtmRevised.importBuiltInCoreDisciplinesCatalog()
game.vtmRevised.importBuiltInBloodMagicCatalog()
game.vtmRevised.importBuiltInRitualCatalog()
game.vtmRevised.importBuiltInWeaponCatalog()
game.vtmRevised.importBuiltInClanCatalog()
game.vtmRevised.importBuiltInBackgroundCatalog()
game.vtmRevised.importBuiltInMeritsFlawsCatalog()
game.vtmRevised.importBuiltInMoralityCatalog()
```

## Разработка

### Проверка каталогов

```bash
npm run validate:catalogs
npm run check
npm run prerelease
```

Валидатор проверяет JSON-валидность каталогов, уникальность сущностей, связи кланов с дисциплинами, связи сил с родительскими дисциплинами, карточки Путей и Дорог, локализацию способностей, наличие файлов из `system.json`, наличие предзагружаемых шаблонов и корректность installable manifest.


## Installable manifest

Foundry устанавливает систему через raw manifest:

```text
https://raw.githubusercontent.com/ercaslehon/vtm-revised-foundry-system/main/system.json
```

В `system.json` должны быть поля:

```json
{
  "url": "https://github.com/ercaslehon/vtm-revised-foundry-system",
  "manifest": "https://raw.githubusercontent.com/ercaslehon/vtm-revised-foundry-system/main/system.json",
  "download": "https://github.com/ercaslehon/vtm-revised-foundry-system/releases/download/v10.0.11/vtm-revised-v10.0.11.zip"
}
```

Ссылка вида `https://github.com/.../blob/main/system.json` не подходит. GitHub отдаёт HTML-страницу, а Foundry ждёт чистый JSON.

## Что не является целью проекта

Проект не претендует на официальный цифровой перенос всех книг VtM.

Справочники внутри системы являются игровыми выжимками, техническими карточками и удобными подсказками для домашнего использования. Для спорных правил, точных формулировок и официального текста следует обращаться к оригинальным книгам.
