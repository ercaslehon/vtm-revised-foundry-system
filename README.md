# Vampire: The Masquerade Revised - Unofficial System v9.2

Эталонная версия Foundry VTT системы для **Vampire: The Masquerade Revised**.

Эта сборка зафиксирована как базовая контрольная точка **v9.2**. От неё стоит вести дальнейшую разработку, делать git-теги и новые feature-ветки. Потому что zip-архивы без версий, как известно, размножаются в темноте и портят людям жизнь.

## Статус

- Package ID: `vtm-revised`
- Version in `system.json`: `9.5.0`
- Foundry compatibility: minimum `13.347`, verified `14`
- Назначение: dev/домашняя система для VtM Revised

## Основные возможности

- Лист персонажа-вампира.
- Портрет персонажа через Foundry File Picker.
- Ограничения поколения: максимум Черт, запас крови, трата крови за ход.
- Натура и Маска с карточками описания.
- Справочник кланов и карточки кланов.
- Одиночные и комбинированные броски d10.
- Ручная сложность броска.
- Штрафы от здоровья к пулу.
- `10 = 2 успеха`, `1 = -1 успех`.
- Здоровье через чекбоксы со штрафами.
- Дисциплины, силы дисциплин и карточки дисциплин.
- Базовые дисциплины 1-5 и расширение 6+.
- Магия крови и пути:
  - Тауматургия;
  - Колдовство;
  - Некромантия;
  - Темная Тауматургия;
  - Чародейство Ассамитов;
  - Чародейство Сеттитов.
- Ритуалы отдельным блоком на всю ширину листа.
- Каталог ритуалов Тауматургии 1-10 уровней.
- Оружие и урон.
- Достоинства, Недостатки и Дополнения с карточками, описаниями и эффектами.
- Пошаговый мастер создания персонажа.
- Визуальные d10 dice effects, экспериментально.

## Установка

Скопируй папку `vtm-revised` в каталог систем Foundry:

```text
FoundryUserData/Data/systems/vtm-revised
```

После этого полностью перезапусти Foundry VTT и создай мир на системе:

```text
Vampire: The Masquerade Revised - Unofficial System
```

## Импорт встроенных каталогов

После установки в мире можно выполнить в консоли Foundry:

```js
game.vtmRevised.importBuiltInClanCatalog()
game.vtmRevised.importBuiltInDisciplineCatalog()
game.vtmRevised.importBuiltInBloodMagicCatalog()
game.vtmRevised.importBuiltInRitualCatalog()
game.vtmRevised.importBuiltInWeaponCatalog()
game.vtmRevised.importBuiltInBackgroundCatalog()
game.vtmRevised.importBuiltInMeritsFlawsCatalog()
```

## Полезные команды

```js
// Импорт персонажа из JSON-чарника
game.vtmRevised.importJsonText(jsonText)

// Синхронизация уже добавленных ритуалов первого персонажа из каталога
game.vtmRevised.syncFirstActorRitualsFromCatalog()

// Применить ограничения поколения к первому персонажу
game.vtmRevised.applyFirstActorGenerationCaps()

// Тест визуальных кубов
game.vtmRevised.showDiceTest()

// Стиль кубов
game.vtmRevised.setDiceStyle("camarilla")
game.vtmRevised.setDiceStyle("sabbat")
game.vtmRevised.setDiceStyle("none")
```

## Правовая оговорка

Это неофициальная fan-made система для Foundry VTT. Она не связана с White Wolf, Paradox Interactive или Foundry Gaming LLC. Тексты справочников внутри системы являются игровыми выжимками и техническими карточками для домашнего использования, а не официальной публикацией правил.

Патч локализации: lang/ru.json и lang/en.json теперь содержат не только плоские ключи, но и вложенную структуру для Foundry VTT v14. Это исправляет отображение строк вида `VTM_REVISED.*` в листе персонажа.

## v9.5.2 Vampire UI Layout Refresh

Сборка делает визуальный слой ближе к целевому интерактивному VTM-чарнику: пересобраны верхний блок персонажа, панель броска, трекеры, сетка основного листа и правая колонка справочных/боевых блоков. Кнопки открытия справки с иконкой книги сохранены для клана, архетипов, Пути/Дороги, дисциплин, ритуалов, достоинств, недостатков и дополнений. Механические action-классы и data-атрибуты сохранены.

## v9.5.3 Vampire UI Shell

Сборка делает лист ближе к целевому интерактивному VTM-дизайну: верх листа перестроен как портретная колонка слева и командный центр справа, трекеры и бросок подняты в верхний shell, ширина листа ограничена для нормальной композиции на широких мониторах, а числовые черты получили декоративные pips без удаления исходных input-полей. Кнопки справки с иконкой книги сохранены.

## v9.5.4 Vampire UI Fix

Исправлена визуальная стабильность shell-макета v9.5.3: верхний блок стал компактнее, панели больше не растягиваются как проклятая простыня, трекеры и тело листа приведены к более плотной сетке. Точки черт теперь строятся динамически от лимита поколения (`generationCaps.traitMax`): для 4 поколения отображается 9 точек, для 3 поколения 10. Кнопки справки-книжечки и механические action-классы сохранены.

## v9.5.5 Actual Tabs

Вкладки листа теперь реально работают. Раздел `Основное` содержит характеристики персонажа: атрибуты, добродетели и способности. Раздел `Силы` содержит дисциплины, отдельные силы, пути магии крови и ритуалы. Раздел `Бой` убран из навигации. Раздел `Профиль` содержит дополнения, достоинства, недостатки, экипировку, Натуру/Маску, профиль, историю, цели, связи, имущество и заметки. Оружие не удалено из системы, а спрятано в сворачиваемый блок профиля, чтобы не ломать существующий функционал.

## v9.5.6 Tabs Polish

Исправлены замечания по вкладкам и читаемости: из вкладки `Силы` убран отдельный блок `Сила дисциплины`, названия дисциплин больше не обрезаются до пары букв, блок здоровья стал полноразмерным и человекочитаемым. При переключении вкладок лист сбрасывает внутреннюю прокрутку вверх, чтобы содержимое не выглядело съехавшим под верхний блок. Функциональные классы, предметы, карточки справки и механика не удалены.

## v9.5.7 Tabs Position Fix

Вкладки `Основное`, `Силы`, `Профиль` вынесены из прокручиваемого тела листа и теперь находятся отдельной строкой между верхним блоком персонажа и содержимым вкладок. Это убирает наложение вкладок на здоровье/ресурсы и предотвращает уезд навигации вверх при переключении вкладок. Прокручивается только содержимое активной вкладки.

## v9.5.8 Scroll From Health

Верхний блок теперь содержит только портрет и основные данные персонажа. Блоки здоровья, ограничений поколения, броска и вкладок вынесены ниже и занимают всю ширину листа, включая область под портретом. Прокручиваемая зона начинается с блока `Здоровье`, поэтому рабочая область больше не ограничена маленьким окном под вкладками.

## v9.5.9 Top Shell Fix

Исправлена ошибка разметки v9.5.8: прокручиваемая зона была случайно вложена внутрь верхнего shell, из-за чего верхний блок съедал всю высоту листа и скрывал здоровье, бросок и вкладки. Теперь top-shell закрывается корректно, а прокрутка снова начинается с блока `Здоровье`.

## v9.5.10 Resources In Header

Ограничения поколения и ресурсные блоки `Кровь`, `Сила воли`, `Человечность/Путь`, `Опыт` перенесены в свободную область верхнего блока под identity-полями. Прокручиваемая зона теперь начинается только с блока `Здоровье`, а дальше идут бросок, вкладки и содержимое активного раздела.

## v9.5.11 Roll Builder Grouped Selects

В панели броска списки выбора параметров теперь сгруппированы по тем же разделам, что и в диалоге броска по нажатию на кубик: физические, социальные и ментальные атрибуты, а также таланты, навыки и знания. Дополнительно сохранены группы добродетелей и ресурсов, если они используются для комбинированных бросков.

## v9.5.12 Glowing Trait Pips

Точки атрибутов и способностей теперь визуально отражают значение числового поля: заполненные точки светятся и мягко мерцают красным. Добавлен live-update при изменении числового поля, чтобы точки загорались сразу, не дожидаясь полного перерендера листа.

## v9.5.13 Health Alignment Polish

Причесан блок здоровья: вместо неровного нативного чекбокса используется аккуратный кастомный индикатор, карточки ранений лучше выровнены по вертикали, а отмеченные состояния читаются чище и выглядят цельнее.

## v9.5.14 Health Compact Selected State

Блок здоровья возвращён к более компактному виду: карточки снова уже, визуальные галочки убраны полностью, а выбор состояния показывается только подсветкой самой карточки.

## v9.5.15 Health No Checks

В блоке здоровья полностью убраны визуальные чекбоксы и галочки. Состояние здоровья теперь отображается только подсветкой выбранной карточки. Карточки сделаны компактнее по высоте и ширине.

## v9.5.16 Blood Magic Names Polish

Исправлена читаемость путей магии крови и ритуалов: названия больше не должны обрезаться до `За...` / `П...`, строки получили больше места, рейтинг и кнопки закреплены справа. На узких окнах блоки магии крови переходят в одну колонку.

## v9.5.17 Blood Magic Full Row

Для путей магии крови и ритуалов название теперь занимает всю первую строку карточки. Рейтинг и кнопки вынесены на вторую строку справа, поэтому названия больше не должны ломаться на странные короткие переносы.

## v9.5.18 Clickable Item Name No Background

У кликабельных названий предметов (включая пути магии крови и ритуалы) убрана кнопочная подложка. Кликабельным остаётся сам текст названия, но без красного прямоугольника под ним.

## v9.5.19 Blood Magic Glowing Pips

Под названиями путей магии крови и ритуалов добавлены сияющие кружки рейтинга по аналогии с атрибутами и способностями. Для путей используется максимум по поколению, для ритуалов — шкала до 10.

## v9.5.20 Discipline Header Plus Alignment

В блоке дисциплин на вкладке «Силы» кнопка добавления перенесена вправо и выровнена по одной линии с заголовком «Дисциплина».

## v9.5.21 Experience Available Only

В блоке «Опыт» на листе персонажа оставлено только поле «Доступно». Поле стало редактируемым напрямую. Поля «Всего» и «Потрачено» скрыты из интерфейса, а автопересчёт доступного опыта от них отключён.

## v9.5.22 Tab Persistence Fix

Исправлен сброс вкладки на «Основное» после нажатия кнопок во вкладках «Силы» и «Профиль». Активная вкладка теперь хранится в экземпляре листа и передаётся в шаблон при перерендере.

## v9.5.23 Discipline Add Dialog

Добавление дисциплины на вкладке «Силы» переведено на отдельное диалоговое окно по аналогии с добавлением путей магии крови. Окно позволяет выбрать дисциплину из импортированного каталога и сразу задать её стартовый уровень.

## v9.5.24 Discipline Catalog Dedup

В диалоге добавления дисциплины список дисциплин из каталога теперь очищается от дублей по названию. Если в мире импортировано несколько одинаковых карточек дисциплины, в выпадающем списке будет показана только одна.

## v9.5.25 Discipline Full Catalog

Диалог добавления дисциплины теперь собирает список не только из импортированных Item-карточек мира, но и из базового системного списка `VTM_REVISED.disciplineOptions`. После объединения список дедуплицируется по названию. Поэтому в выпадающем списке должны появиться дисциплины вроде Стремительности, Доминирования, Химерии и других, даже если их карточки не были отдельно импортированы в директорию предметов.

## v9.5.26 Discipline Dialog Shows Known

Диалог добавления дисциплины теперь показывает полный список дисциплин, включая уже добавленные персонажу. Уже известные дисциплины отображаются в списке с пометкой «уже есть» и недоступны для повторного выбора. Это убирает ложное ощущение, что дисциплины вроде Доминирования, Стремительности или Химерии отсутствуют в каталоге.

## v9.5.27 Discipline Glowing Pips

Под названиями дисциплин на вкладке «Силы» добавлены сияющие точки рейтинга по аналогии с атрибутами, способностями и путями магии крови. Название дисциплины располагается сверху, под ним — светящиеся точки, а справа остаются рейтинг и action-кнопки.

## v9.5.28 Discipline Pips Centered

На вкладке «Силы» сияющие точки дисциплин теперь центрируются по строке вместо левого прилипания под названием. Это убирает ощущение пустого провала в середине строки и делает список дисциплин визуально собраннее.

## v9.5.29 Blood Magic Rows Centered

В строках путей магии крови и ритуалов основной контент теперь выровнен по горизонтальному центру: название и сияющие точки располагаются ровнее и визуально собраннее. Рейтинг и action-кнопки справа сохранены.

## v9.5.30 Blood Magic Vertical Center

В строках путей магии крови и ритуалов восстановлена нормальная левая компоновка основного контента. Название и сияющие точки выровнены по левому краю и центрированы по вертикали внутри строки. Рейтинг и action-кнопки справа тоже выровнены по вертикальному центру.

## v9.5.31 Readable Chat Cards

Исправлена читаемость карточек в чате Foundry. Чат-карточки больше не зависят от CSS-переменных листа персонажа: цвета текста, заголовков, кубов, результатов и карточки изменения ресурсов заданы явно.

## v9.5.32 Weapons Moved To Main

Блок «Оружие и урон» перенесён из вкладки «Профиль» во вкладку «Основное». Теперь он располагается сразу под блоком «Способности», чтобы боевые и часто используемые игровые элементы были ближе к основному чарнику.

## v9.5.33 Correct Revised Abilities

Список способностей приведён к актуальному листу VtM Revised, который используется в проекте. В каждой группе теперь по 10 способностей в правильном порядке: Таланты — Атлетика, Бдительность, Драка, Запугивание, Красноречие, Лидерство, Уличное чутьё, Хитрость, Шестое чувство, Эмпатия; Навыки — Вождение, Воровство, Выживание, Исполнение, Обр. с животными, Ремесло, Скрытность, Стрельба, Фехтование, Этикет; Знания — Гум. науки, Ест. науки, Законы, Информатика, Медицина, Оккультизм, Политика, Расследование, Финансы, Электроника.

## v9.5.34 Portrait Viewer

Клик по портрету персонажа теперь открывает большую картинку персонажа в отдельном окне. Кнопка с иконкой изображения по-прежнему отвечает за смену портрета через FilePicker. Просмотр портрета доступен и игрокам без прав редактирования листа.
